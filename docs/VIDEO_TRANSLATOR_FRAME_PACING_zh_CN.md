# 视频翻译：帧处理节奏（Frame Pacing）

## 目标

在保证字幕更新及时的前提下，控制检测/OCR/翻译开销，避免每帧都跑全流程导致速度过慢。

## 核心参数

- `sample_every_frames`（每 N 帧处理一次）
  - N 小：更新更及时，计算更重
  - N 大：更快，但短字幕可能漏检
- 时序平滑/缓存复用
  - 在未重算帧上复用上次结果，减少闪烁和抖动

## 实践建议

- 动画/快节奏字幕：N=1~3
- 一般影视：N=4~8
- 静态字幕或低要求场景：N=10+

## 诊断

若出现“字幕延迟出现/提前消失”：
1. 降低 N；
2. 检查 FPS 读取是否正确；
3. 对比 SRT 时间轴与烧录结果是否一致。
